# Hyperreal Stone Rendering 技術選定レポート

本レポートは、現実と区別がつかないレベルの石を、構造的正確性（硬さ、割れ方、内部構造の一貫性）を保ちながらリアルタイムレンダリングするために必要な技術スタック全体を、基盤データ構造からレンダリングパイプラインまで網羅的に体系化したものである。

**前提条件:**

- 目標: 写真と区別不可能なレベルの石のレンダリング
- 計算バジェット: 2030年以降のGPUを想定し、強い制約を課さない
- 正確性: 物理的・地質学的に正しい挙動を優先
- インタラクション: ゲームオブジェクトとして認識され、選択・移動・破壊が可能
- 植生統合: 石の隙間から苔や植物が生え、石と植生がそれぞれ独立に識別可能

## 1. 基盤設計

### 1.1 権威的データ構造: 階層的スパースボリュメトリック場

石を「表面」ではなく「体積」として表現することが、本アーキテクチャの根幹である。表面だけを持つメッシュベースの表現では、破壊時の断面表現、内部構造の一貫性、物理シミュレーションとの統合が構造的に不可能になる。

**選定技術: NanoVDB ベースの階層的スパースボクセル場**


| 選択肢                          | 長所                                            | 短所                      | 選定理由              |
| ---------------------------- | --------------------------------------------- | ----------------------- | ----------------- |
| Sparse Voxel Octree (SVO)    | LODとの親和性、階層的メタデータ格納                           | ランダムアクセスが64-treeに劣る場合あり | 階層構造が自然だが、単体では不十分 |
| NanoVDB                      | GPU全API対応、キャッシュフレンドリー、業界標準、NeuralVDBによる100倍圧縮 | 静的トポロジー（値は変更可、構造は不可）    | GPU性能とエコシステムが圧倒的  |
| Neural Implicit (Neural SDF) | 極めてコンパクト、連続的                                  | 編集が困難、推論コスト             | 圧縮レイヤーとして補助的に使用   |


**推奨アーキテクチャ: 二層構造**

```
Layer 1: NanoVDB スパースグリッド (権威的構造層)
├── 粗い解像度で石の形状 SDF を格納
├── per-voxel マテリアル記述子を格納
└── 編集・破壊操作の対象

Layer 2: Neural / Procedural ディテール層
├── Layer 1 のマテリアル記述子を条件として
├── サブボクセル解像度の表面ディテールを生成
└── レンダリング時にのみ評価（権威的状態には含まない）
```

**Rationale:** NanoVDB は NVIDIA が開発した OpenVDB の GPU 最適化版であり、CUDA, Vulkan, HLSL, GLSL の全てに対応する。メモリレイアウトがデフラグメント済みでキャッシュフレンドリーなため、CPU上でも高速にトラバース可能。NeuralVDB（2024年発表）により、既に圧縮済みの VDB データに対してさらに10〜100倍の圧縮が可能であることが実証されている（Disney Cloud データセット: 1.5GB → 25MB）。これにより、per-voxel に豊富なマテリアル情報を格納しても、ストレージ・メモリの問題は技術的に解決可能である。

### 1.2 Per-Voxel マテリアルテンソル場

各アクティブボクセルに以下のマテリアル記述子を格納する:


| フィールド                 | 型         | 説明                      |
| --------------------- | --------- | ----------------------- |
| `mineral_id`          | uint8     | 鉱物種（石英、長石、黒雲母、輝石、方解石 等） |
| `crystal_orientation` | float16×3 | 結晶軸の主方位（劈開面方向の決定に使用）    |
| `porosity`            | float16   | 空隙率（風化シミュレーション、透過光に影響）  |
| `microcrack_density`  | float16   | 微小亀裂密度（破壊閾値、表面粗さに影響）    |
| `weathering_degree`   | float16   | 風化度（0 = 新鮮、1 = 完全風化）    |
| `grain_size`          | float16   | 粒径（テクスチャスケール、光学応答に影響）   |


**Rationale:** これらのフィールドは、以下の全てのサブシステムの入力として機能する:

- **テクスチャ生成**: mineral_id → 色・反射率、crystal_orientation → 異方性パターン、grain_size → テクスチャスケール
- **破壊シミュレーション**: crystal_orientation → 劈開面、microcrack_density → 破壊閾値、porosity → 強度
- **風化シミュレーション**: porosity → 水分浸透、mineral_id → 化学溶解度
- **光学応答**: mineral_id → 屈折率・複素誘電率、porosity → SSS 散乱係数

つまり、一つの統一されたマテリアル場から、外観・物理・破壊の全てが導出される。これが「構造的正確性」を保証する仕組みである。

### 1.3 形状表現: SDF + Dual Contouring

石の形状は Signed Distance Field (SDF) として NanoVDB グリッド内に格納する。表示用のメッシュは Dual Contouring によって抽出する。

**なぜ SDF か:**

- 任意のトポロジー（凹部、オーバーハング、穴）を表現可能
- CSG 演算（smooth union, subtraction）が自然に行える → 破壊操作と直結
- 勾配から法線が解析的に得られる
- LOD 制御が平滑化操作として自然に実現

**なぜ Dual Contouring か:**

- シャープなエッジ（劈開面、破断面）を保存可能（Marching Cubes では不可能）
- Hermite データ（交差点 + 法線）を活用し、サブボクセル精度で表面を配置
- Neural Dual Contouring (NDC, 2022) および Self-Supervised Dual Contouring (SDC, CVPR 2024) により、学習ベースの品質向上が可能
- Occupancy-Based Dual Contouring (ODC, 2024) により、SDF が不完全な領域でも動作可能

抽出されたメッシュは**レンダリングプロキシ**であり、権威的データではない。全ての編集操作は SDF + マテリアル場に対して行い、メッシュは事後的に再抽出する。

### 1.4 ゲームオブジェクトモデル: Entity-Component System

```
Entity: Stone_001
├── VolumetricBody        // NanoVDB場（SDF + マテリアルテンソル）
├── RigidBody             // 物理: 質量、慣性テンソル、反発係数
├── RenderProxy           // 抽出メッシュ + マテリアルバインド
├── Interactable          // 選択、持ち上げ、配置のインターフェース
├── FractureState         // 亀裂ネットワーク、構造健全性指標
└── Children[]
    ├── Entity: Moss_001
    │   ├── VegetationBody    // L-system 状態
    │   ├── RenderProxy       // 植生メッシュ
    │   └── GrowthState       // 成長パラメータ、根の浸透状態
    └── Entity: Fern_001
        ├── ...
```

**Rationale:** ECS を採用する理由:

- **合成による拡張**: 石と植生が異なるコンポーネント集合を持つため、継承階層よりも柔軟
- **データ指向設計**: コンポーネントが SoA (Structure of Arrays) レイアウトで格納され、キャッシュ効率が高い
- **並列処理**: 同種コンポーネントのバッチ処理がGPUフレンドリー
- **階層関係**: 石を親、植生を子とすることで、石を移動すると植生も追従し、かつ個別の選択・編集が可能

Flecs (C99/C++17) や Astra (C++20, 1.05ns/entity iteration @10K) のような高性能 ECS 実装が存在し、実績がある。

## 2. 表面テクスチャ技術マッピング

石の外観を構成する全ての視覚的要素を分解し、各要素に最適な技術を割り当てる。重要なのは、全ての技術が Section 1 で定義したマテリアルテンソル場をパラメータ源として使用する点である。これにより、外観と構造の一貫性が保証される。

### 2.a 鉱物分布パターン（マクロスケール）

**対象:** 花崗岩の斑状組織、玄武岩の均質な暗色、片麻岩の縞状構造 等

**選定技術:**

1. **3D Worley Noise（セルラーノイズ）** — 結晶粒の基本形状生成
2. **Voronoi 結晶粒モデル** — 粒界の生成と鉱物 ID の割り当て
3. **Phase-Field 結晶成長シミュレーション** — 高忠実度が必要な場合のオフライン生成

**技術詳細と Rationale:**

Worley noise は3D空間内のランダムな特徴点までの距離を計算し、セルラーパターンを生成する。これは鉱物の結晶粒界と本質的に同じ幾何学的構造（Voronoi 分割）を持つ。各 Voronoi セルに mineral_id を確率的に割り当てることで、花崗岩のような多鉱物岩の粒状組織を直接的に表現できる。

- 花崗岩: Worley セルに石英(30%)、長石(50%)、黒雲母(20%) を確率的に配置
- 玄武岩: 非常に小さい grain_size + 均質な mineral_id 分布
- 片麻岩: Gabor noise による異方性変調を追加（Section 2.b で詳述）

Phase-field 結晶成長シミュレーションは、メルトからの結晶化過程を物理的に正確にモデル化する手法で、粒界の形状、結晶サイズ分布 (CSD)、優先方位 (CPO) を再現できる。計算コストが高いためオフラインで実行し、結果をマテリアル場に書き込む。

**なぜ単純なテクスチャマッピングではなく Worley/Voronoi か:**
テクスチャマッピングは2D画像を表面に貼り付けるだけであり、石を割った時の断面には対応できない。3D Worley noise は体積的に定義されるため、任意の断面で一貫した鉱物分布パターンが得られる。これが構造的正確性の核心である。

### 2.b 層構造・縞模様

**対象:** 堆積岩の地層、片麻岩の葉理、大理石の脈状模様

**選定技術:**

1. **Reaction-Diffusion シミュレーション（Liesegang パターン）** — 自己組織化縞模様
2. **Gabor Noise** — 異方性・指向性のある層状パターン
3. **堆積シミュレーション** — 堆積岩の地層構造

**技術詳細と Rationale:**

Liesegang パターンは、反応拡散方程式の遠平衡条件で自発的に形成される周期的な縞・リング・多角形パターンであり、ゼブラロック、縞状鉄鉱床、オービキュラー花崗岩などに見られる。Phase-field シミュレーションにより、流体の空間分布と幾何学的不均質性から最終パターン形態を制御できることが実証されている（2025年研究）。

Gabor noise は方位、主周波数、帯域幅をパラメトリックに制御可能なプロシージャルノイズであり、異方性フィルタリングを本質的にサポートする。crystal_orientation テンソルから Gabor noise のカーネル方位を導出することで、結晶学的に正しい方向性を持つ縞模様を生成できる。2024年に Blender に実装されたバージョンでは、C1連続性（バンプマッピングに適する）とHann窓による高速収束が実現されている。

**なぜ Gabor noise か（Perlin noise ではなく）:**
Perlin noise は等方的であり、方向制御が困難。堆積岩の層構造や片麻岩の葉理は本質的に異方的であり、方位パラメータを持つ Gabor noise が理論的に正しい選択である。

### 2.c 風化・摩耗

**対象:** 水流による研磨（川石の丸み）、凍結融解による角張った破砕、化学的溶解によるピッティング

**選定技術:**

1. **Agent-Based 風化シミュレーション** — 凍結融解サイクル、水流研磨
2. **Arenite フレームワーク** — 砂岩特化の物理ベース侵食シミュレーション
3. **パーティクルベース侵食** — 水流・風の効果

**技術詳細と Rationale:**

2024年の研究で、高山環境の凍結融解風化を物理的にシミュレートする手法が発表された。既存の3Dモデルに対して風化サイクルを適用し、時間経過による形状変化を生成する。Blender アドオンとして実装済み。

Arenite（SIGGRAPH 2025）は GPU 加速された砂岩構造シミュレータで、結合力、風による侵食、水流による侵食、粒子堆積を統合的にモデル化する。アーチ、フードゥー、ビュートといった複雑な地形構造を単純な初期条件から5分以内に生成可能。

パーティクルベース侵食（2024年）は、ハイトフィールド、ボクセルグリッド、マテリアルレイヤー、暗黙的地形を含む多様な地形表現に対して動作する汎用手法。侵食を「地形の変更」と「物質の輸送」に分離し、独立したパーティクルで処理することで効率的な並列計算を実現する。

**マテリアル場との接続:**
風化シミュレーションの入力は porosity（水分浸透性）と mineral_id（化学溶解度）であり、出力は weathering_degree と microcrack_density の更新、および SDF の局所的変形である。全てがマテリアルテンソル場を介して一貫している。

### 2.d 微細表面ディテール（マイクロジオメトリ）

**対象:** 微小な凹凸、結晶面のステップ、風化によるピット、研磨痕

**選定技術:**

1. **Nanite テッセレーション + プロシージャルディスプレイスメント** — サブミリメートルの表面形状
2. **高解像度ノーマルマップ** — ピクセルレベルの法線摂動
3. **Procedural Multiscale Geometry Modeling** — 暗黙関数によるマルチスケール粒子構造

**技術詳細と Rationale:**

Nanite テッセレーションは UE 5.4（2024年）で出荷された機能で、メッシュを動的にテッセレーションし、スカラーディスプレイスメント場で変位させる。ジオメトリを明示的に格納せず、ディスプレイスメント値のみを保存するため、データ効率が極めて高い（頂点あたり1値 vs フル座標の5値以上）。プロシージャルテクスチャリングと組み合わせると、格納コストなしで無限解像度のマイクロジオメトリが実現可能。

Procedural Multiscale Geometry Modeling（2025年）は、暗黙関数を使用してマクロからマイクロスケールまでの粒子構造をオンザフライで生成する。粒径、形状、密度、分布、方位を制御可能で、空間的に変動する媒質や粒子凝集をモデル化できる。

**マテリアル場との接続:**
ディスプレイスメントの振幅と周波数は grain_size と mineral_id から導出する。石英は平滑な貝殻状断口、長石は劈開面に沿った階段状のステップ、黒雲母は薄い層状の剥離パターン — これらは全て mineral_id に条件付けされたプロシージャル関数で表現可能。

### 2.e 結晶面の光学応答

**対象:** 花崗岩の雲母の煌めき、石英の硝子光沢、劈開面の鏡面反射

**選定技術:**

1. **Discrete Stochastic Microfacet Model (Glint Rendering)** — 結晶面の煌めき
2. **Position-Normal Manifold** — 高解像度ノーマルマップ上のグリント
3. **薄膜干渉 BRDF (Belcour モデル)** — 鉱物コーティングの虹色干渉

**技術詳細と Rationale:**

Glint rendering は、連続的なマイクロファセット分布を離散的な粒子分布に置き換えることで、結晶面の煌めきを再現する。2024年の研究では、面光源下でのリアルタイムグリントレンダリングが実現され、線形変換コサイン (LTC) と二項分布カウンティングモデルにより最小限のオーバーヘッドで既存レンダリングフレームワークに統合可能。2025年7月の研究では、動的環境マップ下でのイメージベースドライティングにも拡張され、滑らかなマテリアルの約2倍のメモリで動作する。

Position-Normal Manifold（2025年5月）は、高解像度ノーマルマップ表面でのグリント NDF を多様体ベースで定式化し、NDF 積分をメッシュ交差問題として再定式化することで、従来の数値近似より一桁高速な厳密解を達成する。

薄膜干渉 BRDF は、鉱物表面の薄い酸化被膜や風化層が引き起こす虹色の干渉効果をモデル化する。Belcour モデルは RGB レンダリングパイプラインでのスペクトルエイリアシング問題を解決し、Unity HD RP、Unreal、Blender に既に統合されている。

**マテリアル場との接続:**
mineral_id から鉱物の屈折率と結晶系を決定し、grain_size からグリントの粒度を、weathering_degree から薄膜の厚さを導出する。

### 2.f 色彩・空間変動 BRDF

**対象:** 鉱物種に応じた色の空間的変動、風化による色変化、湿潤状態での色の変化

**選定技術:**

1. **条件付き拡散モデルによる SVBRDF 生成** — マテリアル場を条件として高品質マテリアルマップを生成
2. **Inverse Rendering (MatDecompSDF)** — 実写画像からの PBR マテリアル分解
3. **NeuMaDiff** — ニューラルフィールドベースのマテリアル合成

**技術詳細と Rationale:**

ここが拡散モデルの最適な活用場所である。

ReflectanceFusion（2024年）は、Stable Diffusion 2 に特化型の ReflectanceUNet モジュールを組み合わせ、約20万の合成マテリアルで学習された SVBRDF マップ（albedo, normal, roughness, metallic）をテキストプロンプトから生成する。物理パラメータによるカスタマイズとマテリアル属性の直接編集が可能。

**拡散モデルを「ここで」使う理由:**

先述の通り、拡散モデルは「なぜそうなるか」を理解していない。しかし、本アーキテクチャでは「なぜ」はマテリアルテンソル場が既に持っている。拡散モデルの役割は、そのマテリアル場の条件下で「見た目のディテール」を生成する**残差補完**に限定される:

```
入力（条件）: mineral_id=石英, weathering_degree=0.3, grain_size=2mm, porosity=0.05
出力: 高解像度 SVBRDF タイル (albedo, roughness, metallic, normal)
```

こうすることで、拡散モデルの弱点（因果理解の欠如）はマテリアル場が補い、拡散モデルの強み（高品質な視覚的ディテールの生成）を最大限に活かせる。

MatDecompSDF（2025年）は、マルチビュー画像からニューラル SDF と PBR マテリアル（albedo, roughness, metallic）を同時最適化する逆レンダリング手法。実在の石のスキャンデータからマテリアルパラメータを抽出し、プロシージャル生成のキャリブレーションに使用できる。

### 2.g 半透明鉱物のサブサーフェススキャタリング

**対象:** 石英の半透明性、方解石の複屈折、大理石の深部散乱、瑪瑙の層状透過光

**選定技術:**

1. **Hybrid ReSTIR Path Tracing + Diffusion Profile** — リアルタイム SSS
2. **Volumetric Path Tracing** — 高精度参照品質
3. **SSS-GS (Gaussian Splatting SSS)** — 学習ベースの高速 SSS

**技術詳細と Rationale:**

SIGGRAPH 2025 で発表された手法は、ReSTIR パストレーシングと拡散プロファイルを組み合わせることで、オフラインのグラウンドトゥルースに従来の拡散プロファイル手法よりも近い結果をリアルタイムで達成する。大理石、蝋、人肌等の半透明マテリアルで有効性が実証済み。

SSS-GS（NeurIPS 2024 採択）は、3D ガウシアンによる明示的表面表現とニューラルボリュメトリック成分を組み合わせ、150 FPS でフォトリアリスティックなリアルタイム SSS レンダリング、リライティング、マテリアル編集を実現する。

**マテリアル場との接続:**
mineral_id から鉱物の吸収係数と散乱係数を導出し、porosity から内部の散乱密度を決定する。石英 (SiO₂) は可視光領域で高い透過率を持ち、含有する微量元素（鉄、チタン等）が色付きの吸収を引き起こす。これらのパラメータは鉱物学的に既知であり、mineral_id からルックアップ可能。

## 3. 構造的正確性

### 3.1 破壊シミュレーション

**対象:** 石を叩いた時の割れ方、劈開面に沿った破断、粒界に沿った割れ

**選定技術:**

1. **Phase-Field Fracture Model** — 連続場ベースの亀裂伝播（主手法）
2. **Anisotropic Peridynamics** — 動的破壊・分岐（補助手法）
3. **Voronoi 破壊 + 鉱物劈開面制約** — 高速近似（ゲームプレイ用フォールバック）

**Phase-Field Fracture が最適な理由:**

Phase-field 法は亀裂を連続場として表現し、離散的な亀裂面の追跡を不要にする。これにより:

- **亀裂分岐が自然に発生**: 明示的な分岐判定ルールが不要
- **SDF/ボクセル場との統合が直接的**: 亀裂場を NanoVDB の追加チャネルとして格納可能
- **異方性の組み込み**: crystal_orientation テンソルから方向依存の破壊靭性を定義し、劈開面に沿った優先破壊を再現
- **計算効率**: XFEM に対して同等精度で20-40%のCPU時間削減（2025年研究）
- **3D適応精細化**: IGA + PHT-splines による適応メッシュ精細化で、亀裂先端近傍のみ高解像度化（2024年研究で3Dプリント岩石の実験検証済み）

**Peridynamics の補助的役割:**

Peridynamics は非局所的な積分定式化により、亀裂の発生・伝播・分岐を統一的に扱える。特に動的破壊（衝撃荷重、爆発的破砕）において phase-field より優れる。GPU並列化により800倍の高速化が達成されている（2025年研究: 100万粒子×4000ステップが5分）。

異方性ペリダイナミクスモデル（2023-2024年）は、等方性・異方性岩石の両方での亀裂伝播をシミュレート可能で、結晶方位による方向依存性を再現する。

**Voronoi 破壊のフォールバック:**

3D Subspring Network Breakable Voronoi (SNBV) モデル（2025年）は、粒界および粒内の損傷、初期微小亀裂ファブリック、不均質性に起因する局所引張を粒子スケールでキャプチャする。計算コストが低く、ゲームプレイ中のリアルタイムフォールバックとして使用可能。鉱物の劈開面方向で Voronoi 分割を制約することで、物理的妥当性を大幅に向上できる。

**破壊後のパイプライン:**

```
衝撃入力 → 応力場計算 → Phase-field/Peridynamics で亀裂場更新
→ 亀裂場が閾値を超えた領域で SDF を分割
→ 新しい表面が露出 → マテリアル場をスライス → 断面の外観が自動的に正しい
→ 分離したフラグメントを新しい Entity として生成
→ 各フラグメントに RigidBody, RenderProxy, VolumetricBody を付与
```

**鍵となるポイント:** 破断面の外観は「生成」する必要がない。マテリアルテンソル場は体積的に定義されているため、新しく露出した断面は場をスライスするだけで、正しい鉱物分布、粒状組織、層構造が自動的に得られる。これが体積的表現を権威的状態とする最大の利点である。

### 3.2 弾性と衝突応答

**選定技術:**

1. **反発係数ベースの衝突応答** — 基本的な跳ね返り/吸収
2. **ボクセル場からの物性パラメータ自動導出** — 材料定数の一貫性
3. **VoMP (Volumetric Mechanical Property) Prediction** — ニューラルネットワークによる体積的物性予測

**三段階モデル:**


| 段階              | 条件                  | 処理                                 |
| --------------- | ------------------- | ---------------------------------- |
| Tier 1: 弾性衝突    | 衝撃エネルギー < 弾性限界      | 反発係数 (CoR) で処理。石同士: 0.2-0.5        |
| Tier 2: 確率的破壊判定 | 弾性限界 < エネルギー < 破壊閾値 | mineral_id 別の破壊靭性とキャリブレーション済み閾値で判定 |
| Tier 3: 破壊実行    | エネルギー > 破壊閾値        | Section 3.1 の破壊シミュレーションを起動         |


**VoMP（2025年、NVIDIA）:**
Geometry Transformer と MatVAE を使用し、ガウシアンスプラット、NeRF、SDF を含む3D表現から体積的な機械的特性（ヤング率、ポアソン比、密度）を予測する。オブジェクトごとの最適化なしにシミュレーション可能なアセットを生成でき、現実的な変形可能物理シミュレーションをサポートする。

**マテリアル場との接続:**
反発係数、破壊靭性、ヤング率は mineral_id と porosity から鉱物学的データベースを参照して導出する。例: 石英 (モース硬度7, CoR≈0.5) vs 方解石 (モース硬度3, CoR≈0.3)。複合岩石（花崗岩等）の場合は、ボクセル場の鉱物分布から有効物性を均質化する。

### 3.3 内部表現

石の内部は、外観として別途「生成」する必要がない。これは本アーキテクチャの最も重要な設計上の帰結である。

**なぜ自動的に正しいか:**

1. マテリアルテンソル場は3D空間全体に定義されている
2. 3D Worley noise による鉱物分布は体積的に一貫している
3. Reaction-diffusion による層構造も体積的に定義されている
4. 風化度は表面から内部へのグラデーション（表面が高、内部が低）

したがって、石を割った時:

- 断面に花崗岩の粒状組織が見える（3D Worley が体積的に定義されているため）
- 断面の鉱物は新鮮（weathering_degree が低い）で、外表面より明るい色を持つ
- 層構造が断面に自然に現れる（reaction-diffusion が3Dで定義されているため）
- 劈開面は結晶方位に沿った平滑な面として現れ、粒界破壊は粗い面を持つ

これは、テクスチャマッピングベースのシステムでは原理的に不可能な特性である。

### 3.4 衝突判定パイプライン

石のインタラクション（持ち上げ、配置、衝突、破壊）を物理的に正しく実現するには、不規則な形状を持つ石同士、および石と環境の衝突判定が高精度かつリアルタイムで動作する必要がある。本エンジンでは石の権威的形状が SDF として定義されているため、SDF を直接活用した衝突判定パイプラインを設計する。

#### 3.4.1 ブロードフェーズ: BVH + GPU RT コア加速

**選定技術: Bounding Volume Hierarchy (BVH)**

| 候補 | 長所 | 短所 | 選定判断 |
|------|------|------|----------|
| Sweep-and-Prune (SAP) | 低オーバーヘッド、ソート済みリストの差分更新が高速 | 軸方向に偏った分布で性能低下、不規則形状の AABB が過大 | 補助的に使用可 |
| Spatial Hashing | 均一分布で高速、実装が単純 | サイズが大きく異なるオブジェクト混在時に非効率 | 石のサイズ多様性に不向き |
| BVH | 不規則サイズ混在に強い、階層的カリングが効果的、GPU RT コアで直接加速可能 | 動的オブジェクトの再構築コスト | **主手法として採用** |

**Rationale:**

石は小さな礫から大きな岩まで桁違いのサイズ差があり、かつ破壊後のフラグメントが動的に増加する。BVH はこのサイズ不均一性と動的オブジェクト数の変動に対して最もロバストに動作する。

さらに、Mochi（2024年）で実証されたように、GPU のレイトレーシングハードウェア（RT コア）を BVH トラバーサルに転用することで、5x〜28x の高速化が達成されている。衝突判定問題をレイトレーシング問題に変換する手法（オブジェクトを BVH にインデックスし、RT コアで階層探索を実行）は、本エンジンが既に Vulkan RT 拡張を使用するレンダリングパイプラインを持つ前提と自然に統合できる。

**AABB 更新戦略:**
- スリープ中の石: AABB 更新をスキップ（ゼロコスト）
- 低速移動中: AABB にマージンを付与し、マージン内の移動では再構築しない
- 高速移動（破壊フラグメント等）: 毎フレーム AABB 再計算 + CCD 起動（Section 3.4.3）

#### 3.4.2 ナローフェーズ: SDF-SDF 直接衝突判定

**選定技術: 一般 SDF-SDF リアルタイム衝突判定（2024年発表）**

本エンジンの石は SDF が権威的形状表現であるため、SDF を直接的に衝突判定に使用するのが設計上最も自然かつ正確である。

2024年に発表された一般 SDF-SDF 衝突判定手法は、任意の SDF 同士の交差を初めてリアルタイムで計算可能にした。以下がその動作原理:

1. **バウンディングボリュームによるセグメント化**: 複雑な SDF を複数の部分領域に分割し、衝突候補を絞り込む
2. **区間演算 (Interval Arithmetic)**: SDF 値の区間を伝播させ、交差が存在しない領域を早期棄却
3. **SDF 勾配ガイド探索**: 交差領域内で SDF の勾配を用いて接触点を効率的に収束
4. **出力**: 貫通深度、接触点座標、接触法線を算出

**なぜ GJK/EPA ではなく SDF 直接評価か:**

| 観点 | GJK/EPA | SDF 直接評価 |
|------|---------|-------------|
| 入力形状 | 凸形状のみ（凹形状は凸分解が必要） | 任意の SDF（凹部、オーバーハング、穴を含む） |
| 前処理 | 凸分解（V-HACD 等）が必要、近似による精度劣化 | 不要（権威的 SDF をそのまま使用） |
| 石への適合性 | 石の不規則な凹凸を凸分解すると大量のハル、精度と性能が劣化 | 石の SDF を直接評価、不規則形状に自然に適合 |
| 権威的データとの一貫性 | メッシュプロキシ経由で間接的 | SDF が権威 → 衝突判定も権威的形状を直接使用 |
| 破壊後の対応 | 新フラグメントの凸分解が必要 | SDF 分割で即座に衝突判定可能 |

**フォールバック:**
SDF が未生成または低解像度のフラグメント（生成直後の破片等）に対しては、Convex Hull + GJK/EPA を暫定的に使用し、SDF 生成完了後に切り替える。

**LOD 統合（Section 6.2 参照）:**
ナローフェーズの SDF 解像度は距離・重要度に応じて段階的に変える:

| 距離/状態 | 衝突形状 | ナローフェーズ手法 |
|-----------|---------|-----------------|
| 遠距離 / スリープ中 | 球 or AABB | ブロードフェーズのみ（ナローフェーズ不要） |
| 中距離 | Convex Hull | GJK/EPA |
| 近距離 / 接触中 | 低解像度 SDF | SDF-SDF 衝突判定（粗いグリッド） |
| 高精度要求（破壊判定等） | 高解像度 SDF | SDF-SDF 衝突判定（細かいグリッド） |

#### 3.4.3 連続衝突判定（CCD）

**対象:** 破壊後の高速フラグメント、投げられた石、爆発的破砕片のトンネリング（薄い壁をすり抜ける現象）防止。

**選定技術:**

1. **Scalable CCD（GPU 並列、区間ベース）** — 主手法
2. **Soft SDF + SCM（Smooth Contact Model, 2025年）** — 高品質オプション

**Scalable CCD:**
GPU 上でブロードフェーズ（スイープアルゴリズム）とナローフェーズ（区間ベース手法）を並列実行する手法。浮動小数点演算での証明可能な正確性を持ち、false negative（すり抜け）がゼロであることが数学的に保証されている。

**Soft SDF + SCM（2025年）:**
Soft Signed Distance Function (SSDF) は任意の閉じた 3D 表面を近似し、楕円体への滑らかな補間を提供する。Soft-Minimum Contact Model (SCM) は接触を交差体積全体に分布する滑らかな力場として表現する。これにより:
- 二階微分可能な力学（反復ソルバー不要）
- 接触数に依存しないランタイム
- 凸分解不要で任意の衝突形状に対応

本エンジンの SDF ベースの権威的表現と直接統合可能であり、2030年の GPU バジェットでは主手法として採用する価値がある。

**適用条件:**
CCD は全オブジェクトに常時適用するのではなく、以下の条件でのみ起動する:

```
if (linear_velocity > ccd_velocity_threshold ||
    angular_velocity > ccd_angular_threshold ||
    object_size < ccd_size_threshold) {
    // CCD を起動
} else {
    // 離散衝突判定で十分
}
```

通常の静止した石や低速移動中の石には離散衝突判定で十分であり、CCD のコストを支払う必要はない。

#### 3.4.4 接触マニフォールド生成

石の安定した接地と物理的に正しい摩擦応答のためには、単一の接触点ではなく、複数の接触点からなる**接触マニフォールド**が必要である。

**選定技術:**

1. **Persistent Contact Manifold** — フレーム間キャッシュ
2. **SSDF ベースの接触力場** — 滑らかな分布接触

**Persistent Contact Manifold:**
フレーム間で接触点をキャッシュし、新しい接触点のみを追加・古い接触点を除去する。これにより:
- 接触解のフレーム間安定性が向上（ジッターの低減）
- 接触点の再計算コストが削減
- 石が地面に静止する際の安定性が保証される（3点以上の接触マニフォールドにより転倒しない）

**SSDF ベースの接触力場（2025年）:**
SSDF と SCM を組み合わせると、離散的な接触点ではなく、交差領域全体に分布する連続的な接触力場を生成できる。不規則な表面を持つ石同士の接触において、離散接触点ベースの手法で発生しがちな不安定性（接触点の急激な切り替え、力の不連続）を回避できる。

**石の不規則形状における安定接地:**
石は底面が平坦ではないため、地面との接触は通常 3〜5 個の凸部に集中する。Persistent Contact Manifold はこれらの接触点を維持し、LOD 遷移時にも急激な接触状態の変化を防ぐ。Section 6.2 の LOD 遷移ポリシー（ヒステリシス、テンポラルブレンド）と組み合わせることで、LOD 変更による石の「浮き」や「沈み」を抑制する。

#### 3.4.5 マテリアル場との接続

衝突判定の結果は、マテリアルテンソル場と以下のように接続する:

**摩擦係数の導出:**
接触点におけるマテリアル場の mineral_id と weathering_degree から、鉱物学的に妥当な摩擦係数を導出する。

| 条件 | 静止摩擦係数 | 動摩擦係数 |
|------|------------|-----------|
| 乾燥した石英表面 | 0.6 - 0.8 | 0.4 - 0.6 |
| 濡れた石英表面 | 0.3 - 0.5 | 0.2 - 0.4 |
| 風化した石灰岩 | 0.5 - 0.7 | 0.3 - 0.5 |
| 苔が付着した表面 | 0.2 - 0.4 | 0.1 - 0.3 |

これらの値は mineral_id、weathering_degree、moisture パラメータから**連続関数**として導出される（ハードカットオフではない）。

**反発係数の導出:**
Section 3.2 の Tier 1 で使用する反発係数 (CoR) は、接触点の両側のマテリアル場から導出する。接触点の mineral_id と porosity から各オブジェクトの CoR を計算し、衝突ペアの有効 CoR を算出する。

**破壊パイプラインへの接続:**
衝突エネルギーが Section 3.2 の Tier 2/3 の閾値を超えた場合、衝突判定で得られた接触点・接触法線・貫通深度を破壊シミュレーション（Section 3.1）に引き渡す:

```
衝突判定（接触点、法線、エネルギー）
→ Tier 判定（弾性限界 vs 破壊閾値）
→ Tier 3 の場合: 接触点を起点として応力場を計算
→ Phase-field / Peridynamics で亀裂場を更新
→ SDF 分割 → 新 Entity 生成
```

**damage_d フィールドとの相互作用:**
損傷が蓄積された領域（damage_d が高い）では、破壊閾値が局所的に低下する。衝突判定で得られた接触点近傍の damage_d を参照することで、「既にひびが入っている箇所を叩くと少ない力で割れる」という物理的に正しい挙動が実現される。

#### 3.4.6 技術選定サマリー

| フェーズ | 技術 | 状態 | Rationale |
|---------|------|------|-----------|
| ブロードフェーズ | BVH + GPU RT コア加速 | 製品レベル + 研究 (Mochi 2024) | サイズ不均一、動的増減に強い。RT コアで追加高速化。レンダリングパイプラインと BVH 共有可能 |
| ナローフェーズ | SDF-SDF 直接衝突判定 | 研究段階 (2024) | 権威的 SDF を直接使用。凸分解不要。不規則な石の形状に自然に適合。破壊後の SDF 分割と即座に統合可能 |
| ナローフェーズ (フォールバック) | Convex Hull + GJK/EPA | 製品レベル | SDF 未生成フラグメントの暫定処理 |
| CCD | Scalable CCD (GPU 並列) + SSDF/SCM | 研究段階 (2024-2025) | 高速フラグメントのトンネリング防止。SSDF は SDF 権威表現と直接統合可能 |
| 接触マニフォールド | Persistent Manifold + SSDF 接触力場 | 製品 + 研究 (2025) | 不規則形状の安定接地。LOD 遷移時の安定性。連続的な接触力で不連続ジッターを回避 |
| マテリアル連携 | mineral_id/porosity/damage_d → 摩擦/反発/破壊閾値 | 設計段階 | マテリアルテンソル場から全パラメータを一貫して導出。構造的正確性の保証 |

## 4. レンダリングパイプライン

### 4.1 ハイブリッドアプローチの全体設計

**基本方針:** ラスタライゼーションで効率的なプライマリビジビリティを確保し、レイ/パスベース手法で物理的に正しい光輸送を実現する。

```
┌─────────────────────────────────────────────────────────┐
│                    フレームパイプライン                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
```

### 4.2 プライマリビジビリティ: Nanite 仮想ジオメトリ

**選定理由:**

Nanite 方式の仮想ジオメトリは、ポリゴン数・ドローコール数の制約を排除し、映画品質のアセットを直接レンダリング可能にする。Visibility Buffer アプローチ（64ビットアトミック操作、ハードウェア+ソフトウェアラスタライゼーションの併用）により、メッシュレットの投影サイズに基づいた自動 LOD 管理が実現される。

Nanite テッセレーション（UE 5.4, 2024年）により、ディスプレイスメントマップからの動的メッシュ生成が可能。石の微細な表面凹凸を、データ格納なしでプロシージャルに生成できる。

**なぜ全面パストレーシングではないか:**

2030年のGPUであっても、全ピクセルの全バウンスをパストレーシングで処理するのはオーバーキルである。プライマリビジビリティ（最初のカメラレイが何に当たるか）はラスタライゼーションが本質的に効率的であり（O(n) vs O(n log n)）、レイトレーシングの恩恵は二次光線以降（反射、GI、影、SSS）で最大化される。

### 4.3 間接照明: ReSTIR + Neural Radiance Cache

**ReSTIR (Reservoir-based Spatiotemporal Importance Resampling):**

ReSTIR は、ピクセル間・フレーム間でサンプルを再利用することで、パストレーシングの品質を大幅に向上させる。

- **ReSTIR DI**: 大量の光源を効率的にサンプリング
- **ReSTIR GI / PMGI**: 間接照明のためのフォトンマッピング + 仮想光源
- **ReSTIR BDPT（2025年）**: 双方向パストレーシングと ReSTIR を組み合わせ、コースティクスや到達困難な光源の処理を大幅改善（バスルームシーンで MAPE: 0.312 vs 単方向 ReSTIR PT の 1.368）

**Neural Radiance Cache:**

Lumen（UE5）で実証されたアプローチ。方向依存の放射照度情報をオクタヒドラルマップにキャッシュし、オンデマンドで取得する。ブルートフォースレイトレーシングと高速なライティング再利用のギャップを埋め、マルチバウンス GI とリフレクションを実現する。

### 4.4 石に特化したレンダリング手法


| 現象       | 手法                                   | 適用条件                                           |
| -------- | ------------------------------------ | ---------------------------------------------- |
| 結晶の煌めき   | Discrete Stochastic Microfacet + LTC | mineral_id ∈ {雲母, 石英} かつ grain_size > 0.5mm    |
| 劈開面の鏡面反射 | Position-Normal Manifold NDF         | 新鮮な破断面 (weathering_degree < 0.1)               |
| 薄膜干渉（虹色） | Belcour Thin-Film BRDF               | weathering_degree > 0.5 かつ mineral_id ∈ {含鉄鉱物} |
| 半透明      | Hybrid ReSTIR-PT SSS                 | mineral_id ∈ {石英, 方解石, 瑪瑙} かつ grain_size > 1mm |
| 濡れた表面    | roughness 低減 + 薄い水膜 F0 変調            | 湿度パラメータ > 0.5                                  |


**Rationale:** 全ての石が全ての光学効果を必要とするわけではない。マテリアル場の mineral_id と weathering_degree に基づいてシェーディングパスを分岐させることで、計算コストを必要な箇所にのみ集中させる。

### 4.5 参照品質モード

開発・キャリブレーション用に、全てのショートカットを無効化した参照品質モードを維持する:

- 全ピクセルでフルスペクトルボリュメトリックパストレーシング
- デノイジングなし
- テンポラルアキュムレーション（数百〜数千 SPP）
- 分光レンダリング（RGB ではなく波長サンプリング）

これにより、ハイブリッドパイプラインの近似品質を常に検証可能にする。

## 5. 植生統合

### 5.1 植生成長シミュレーション

**選定技術:**

1. **Space Colonization Algorithm** — 根・苔の表面成長パターン
2. **L-System** — 茎・葉の構造的成長
3. **Riemannian L-System（2024年）** — 曲面上での成長

**Space Colonization Algorithm の適用:**

従来の L-system が根元から外向きに成長するのに対し、space colonization algorithm は「アトラクションポイント（成長目標）」を先に配置し、そこに向かって枝を伸ばす。これにより:

- 石の亀裂内に「湿度の高い領域」をアトラクションポイントとして設定
- 根がそれらのポイントに向かって自然に成長
- 石の表面形状に制約された成長パスが得られる

Clasping Trees（Eurographics 2024）で実証されたパイプラインでは、メッシュのボクセル化を使用して表面・体積上での space colonization を実行し、ゲームエンジン内で直接プロシージャル生成を行う。

**Riemannian L-System（2024年）:**

リーマン幾何学に拡張された L-system により、曲面上（石の表面）での生物的形態のモデリングが可能。葉脈ネットワーク、曲面上のフィラメント、表面コロニゼーションパターンの生成に適用される。

### 5.2 石の表面特性からの成長条件導出

```
石のマテリアル場から導出:
├── 湿度マップ: porosity × 降雨量 × 表面の凹み度
├── 日照マップ: 表面法線と太陽方向から計算
├── 亀裂マップ: microcrack_density から抽出
└── 栄養素マップ: mineral_id から化学的風化度を推定

これらを成長条件として:
├── 苔: 湿度 > 0.6 かつ 日照 < 0.4 の領域に高密度
├── シダ: 亀裂幅 > 5mm かつ 湿度 > 0.4 の領域に根を配置
└── 地衣類: weathering_degree > 0.3 の表面に薄層として
```

### 5.3 ECS における石と植生の関係

```
Entity: Stone_001
├── VolumetricBody (権威的ボクセル場)
├── RenderProxy (石のメッシュ)
├── Interactable (選択: 石全体)
└── Children[]
    ├── Entity: Moss_Patch_001
    │   ├── VegetationBody (L-system 状態, 被覆密度マップ)
    │   ├── RenderProxy (苔のメッシュ/ビルボード)
    │   ├── Interactable (選択: 苔パッチ個別)
    │   └── SurfaceBinding (親表面の UV 座標/世界座標にバインド)
    └── Entity: Fern_001
        ├── VegetationBody (L-system 状態)
        ├── RenderProxy (シダのメッシュ)
        ├── Interactable (選択: シダ個別)
        └── RootPenetration (石の microcrack_density を局所的に増加)
```

**重要な相互作用:**

- 植物の根は石の microcrack_density を局所的に増加させる（根の浸透による風化促進）
- 石を移動すると子エンティティの植生も追従
- 石を破壊すると、破壊面上の植生は分離したフラグメントに再バインドされる
- 植生は石と独立に選択・削除・編集可能

## 6. ゲームオブジェクト認識

### 6.1 オブジェクト選択とハイライト

**選定技術:**

1. **Stencil Buffer + 拡張メッシュ描画** — オブジェクトアウトライン
2. **Object ID バッファ** — ピクセル単位のオブジェクト識別

**パイプライン:**

G-Buffer パスで object_id を各ピクセルに書き込む。マウスクリック時にピクセルの object_id を読み取り、対応する Entity を特定する。選択された Entity の RenderProxy に対して:

1. Stencil buffer にオブジェクトのシルエットを書き込む
2. わずかに拡大した（法線方向に押し出した）バージョンを描画
3. Stencil テストにより、元のシルエット外のピクセルのみをアウトラインカラーで描画

これにより、オブジェクト重なり部分でも正しいアウトラインが得られる。

### 6.2 階層的衝突判定 LOD


| 距離/状態       | 衝突形状        | コスト         |
| ----------- | ----------- | ----------- |
| 遠距離 / スリープ中 | 球 or AABB   | O(1)        |
| 中距離         | Convex Hull | O(log n)    |
| 近距離 / 接触中   | 低解像度 SDF    | O(n) だが局所的  |
| 高精度要求       | 高解像度 SDF    | O(n) だが限定領域 |


**LOD 遷移ポリシー:**

- ヒステリシス: 一度高精度にしたら一定時間は低精度に戻さない
- スリープ中は接触マニフォールドを凍結
- LOD 変更はベロシティ/トルク閾値が低い時のみ許可
- 接地面の LOD 遷移はテンポラルブレンド（瞬間切り替えではなく漸次移行）

### 6.3 インタラクション


| アクション | 処理                                                    |
| ----- | ----------------------------------------------------- |
| 選択    | object_id バッファから Entity 特定 → Stencil アウトライン           |
| 持ち上げ  | RigidBody を kinematic モードに → Interactable のハンドポイントに追従 |
| 配置    | RigidBody を dynamic モードに戻す → 物理シミュレーションで着地            |
| 破壊    | Section 3.1 の破壊パイプライン → フラグメントを新 Entity として生成         |
| 植生追加  | 石の表面に成長条件を評価 → 条件を満たす位置に植生 Entity を生成                 |
| 植生除去  | 子 Entity を削除 → 石の microcrack_density の植生影響をリセット       |

### 7.4 この設計が要件を満たす理由

**現実と区別がつかないレベル:**

- マテリアルテンソル場から導出される SVBRDF は鉱物学的に正しいパラメータを持つ
- Glint rendering が結晶面の煌めきを再現し、SSS が半透明鉱物の深みを表現する
- Nanite テッセレーションによりサブミリメートルの表面ディテールがピクセルスケールで描画される
- ReSTIR BDPT + Neural Radiance Cache により物理的に正しい光輸送が実現される

**構造的正確性:**

- 体積的マテリアル場が全ての導出の唯一の源泉
- 破断面は場のスライスとして自動的に正しい外観を持つ
- 破壊は鉱物の劈開面と粒界に沿って物理的に正しく進行する
- 風化は鉱物種と空隙率に基づいて差分的に進行する

**ゲームオブジェクトとしての認識:**

- ECS により石と植生が独立したエンティティとして管理される
- Object ID バッファによりピクセル単位の識別が可能
- 階層的衝突 LOD により効率的なインタラクションが実現される
- 破壊後のフラグメントは自動的に新しいゲームオブジェクトになる

**植生の独立識別:**

- 石と植生は ECS 上で親子関係にある別エンティティ
- 独立した選択・編集・削除が可能
- 植生の成長は石の物理的特性から導出され、配置が物理的に妥当
